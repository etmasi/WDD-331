CSS Code Review

We've been reading and talking a lot this week about css organization and architecture. We've only scratched the surface. This is an issue that every developer faces, and there are about as many opinions on the correct way to do it as there are developers...or projects...maybe even developers X projects. That's how many opinions there are ;)

You will eventually develop your own opinions as well. This activity will hopefully help you begin to do that by reviewing the css for a popular project...Bootstrap.

Activity

Based on the reading and discussions of this week about organization and architecture we will review the source code (css) for the Bootstrap framework and discuss what they have done to organize their code.

Step 1

Make sure to complete the Bootstrap installation activity before continuing. You can find the Bootstrap src files in node_modules/bootstrap/scss

Step 2

Review Bootstrap and write-up your thoughts considering the following as you review:

  • Is the styling is de-coupled from the html structure? Try using a few components in a page like button, navbar, or progress bar. (If you visit Css settings in codepen you can quickly add Bootstrap as an External CSS resource…see the Quick-add box) Can you use them by just using classes or do they require very specific markup to work?
  • Which if any of the philosophies we discussed this week they might be using (BEM, Atomic, OOCSS, etc)?
  • Are the folders they use well named?
  • Check out the variables partial. Imagine trying to customize Bootstrap without that settings file! Consider the advantages Sass gives to both the organization and customization of Bootstrap.
  • If you were a developer working on this code and you were asked to work on on something specific...like headline styles...how hard would it be to locate the code and modify it? Would you worry about your changes having unintended consequences after reviewing the class names and architecture of that portion of css?
  • Do you like how it is organized? Would you change anything?

Step 03:  Submit your notes to Ilearn.

Grading

  • Thoughtful review of the code given. 5pts
  • At least 250 words. 5pts
